ADAPTIVE LEARNING SYSTEM — HONEST ASSESSMENT
=============================================
Date: 2026-07-01


1. IS THE SYSTEM ACTUALLY IMPROVING OUTCOMES?
---------------------------------------------
No — and there is currently no way to measure it.

There is no outcome tracking infrastructure in the codebase:
- No before/after comparison of scores per student
- No time-series on progressMetrics (it is a single flat object per subject,
  overwritten each time — no history)
- No A/B tracking, no cohort comparison, no engagement metrics
- improvementTrend (improving/declining/stable) is a stored enum field, but
  nothing in the routes ever sets it — it stays at its default 'stable' forever

To actually measure outcomes you would need, at minimum:
- A scoreHistory: [{date, score, subjectId}] array written every time an
  assignment is graded
- A weekly/monthly job that computes slope over that history and writes
  improvementTrend
- A dashboard query: students who received a generated learning path vs those
  who did not → compare 30-day score delta

None of that exists yet.


2. WHAT DATA IS STORED PER STUDENT, AND WHAT DRIVES PERSONALIZATION?
----------------------------------------------------------------------
Stored in StudentUser:
  demographics, class/section/roll, attendance array, achievements,
  encrypted PII (mobile, email, Aadhaar). No learning-specific fields.

Stored in StudentProgress:

  Field                 | What it holds                                        | Actually written?
  ----------------------|------------------------------------------------------|------------------------------------------
  submissions[]         | assignmentId, score, feedback, status                | Yes — via assignment grading
  progressMetrics[]     | averageScore, totalAssignments, attendanceRate        | Yes — via assignment routes
  weaknessAnalysis[]    | consistencyScore, weakAreas[], recommendedTopics[]   | Only on manual admin API call
  aiLearningPaths[]     | currentTopic, completedTopics[], resources[], %      | Only on manual admin API call
  isWeakStudent         | Boolean                                              | Only after manual analyze-weakness call
  interventionLevel     | low/medium/high/critical                             | Only after manual analyze-weakness call
  improvementTrend      | improving/declining/stable                           | NEVER WRITTEN — always 'stable'

What actually drives personalization today: nothing automatic.
The analysis and path generation are both admin-triggered, manual API calls.
No student action (viewing a material, answering a quiz, completing a topic)
automatically updates any of this.


3. HOW ARE LEARNING PATHS GENERATED? WHAT LOGIC DETERMINES THE LESSON?
-----------------------------------------------------------------------
The generateAILearningPath() function in aiLearningRoute.js is pure hardcoded
if/switch logic — no ML, no LLM, no Qdrant:

  weakAreas[] (passed in manually by the admin)
    → switch(subject) {
        case 'mathematics': if area includes 'Basic Concepts'
          → push hardcoded resource with URL '/learning/math/fundamentals'
      }
    → returns list of resources with static URLs

Subject coverage: only mathematics, physics, chemistry, biology.
Any other subject gets two generic fallback resources pointing to
'/learning/{subject}/review' and '/learning/{subject}/quiz' — URLs that do
not exist anywhere in the frontend or backend.

CRITICAL BUG in analyzeStudentWeakness():
  The subject filter is a commented-out stub:

    const subjectSubmissions = progress.submissions.filter(sub => {
      // You would need to join with Assignment model to get subject info
      return true; // BUG: uses ALL assignments regardless of subject
    });

  So the consistency score for "Mathematics" is computed using every
  assignment the student has ever submitted, across all subjects.


4. SUMMARY — WHAT YOU HAVE VS WHAT ADAPTIVE LEARNING REQUIRES
--------------------------------------------------------------

  Capability                                      | Required | Status
  ------------------------------------------------|----------|---------------------------
  Per-student score history (time-series)         | Yes      | No — flat metric, overwritten
  Automatic re-analysis on new data               | Yes      | No — manual admin trigger
  Learning path from actual curriculum content    | Yes      | No — hardcoded URLs (broken)
  Subject-scoped weakness analysis                | Yes      | No — all subjects mixed (bug)
  Student engagement tracking (time-on-task)      | Yes      | No — not modeled
  Outcome measurement (score delta after path)    | Yes      | No — improvementTrend never set
  LLM/RAG-powered path generation                 | Future   | No — pure switch/if logic

The data model (aiLearningPaths, weaknessAnalysis) is designed for the right
thing, but the engines that should populate it are not built yet.

The AI tutor (Qdrant + Ollama RAG) is the only genuinely functional AI
component right now.
